Introduction

CSS stands for cascading styles sheet. As it's name implies, CSS is a markup language that allows Html elements to be styled. Let's paint a picture, HTML is the structure of a webpage and what gives structure to the human body? Bones. CSS is appearance synomymous to the flesh slapped on the bones while Javascript is the brain, giving life, movements to the body. Get the picture now? We will be learning about some core concept of CSS.

Linking Your CSS

Before we get down to all the good stuff, let us take a look at linking our stylesheet. It's important that styles are linked one way or another to your HTML file. Not many developers will admit that they forget to link their stylesheet and spend hours wondering why the styles aren't applying on the webpage 😆 There are three ways to connect stylesheet to a html file, listed in the order of highly recommended to try to avoid doing this:

External CSS
Every HTML page must include a reference to the external style sheet file inside the <link> element which is a child of the <head> section. If you would use the same stylesheet on another html file, it's recommended that you use the external linking of stylesheet.
Internal CSS
The <style> element which will contain all of your css styling is imbedded inside the <head> section. Both Html and Css is contained in one page. When there are several elements to be styled, it can make your html page bulky and debugging your code would be nightmarish. It's advisable to have your css in a separate, but whatever floats your boat.
Inline CSS
Third and definitely the least recommended way to style html element.

<div style="padding:10px; background-color:blue;">Box1</div>

Selectors

Selectors are used to find or access html elements and style them. There are several kinds of selectors but for the purpose of simplicity, we will stick with the basic selectors.

  • Element name selectors e.g h1, p, div, span, table etc.
  • Class selectors e.g .heading1 will select all element with a class of heading1.
  • Id selectors e.g #bigGuy will select an element with id of bigGuy
  • Attribute selectors e.g [attr="value"]
Let's take a look at some codes

< h1 id="bigGuy" class="heading1"> I am the biggest heading element < h1/>

Let's add a little css in that

h1 { color: red; font-weight: italic; background-color: #ddd; padding: 5px 10px; }

Box Model

Every element in html is a box with four layers: content, padding, border and margin.

  • Content box is where text and images appear
  • Padding is the clear space around the content
  • Border goes around the box in-between the padding and the margin
  • Margin is the space outside the box

If you choose to use the standard border box, then border, margin and padding is added to the width and height of an element. What does this mean? Take a look at the code below

div { background-color: yellow; height: 50px; width: 65px; padding: 20px; margin: 10px 15px; border: 5px solid gray }

You'd expect the width to be 65px and height 50px, right? Nope! The total width would be: 65px + {padding-left(20px)} + {padding-right(20px)} + {border-left(5px)} + {border-right(5px)} = 115px and the height? 100px. If you want to avoid these phemomenon, then include box-sizing: border-box;

Responsive Web Design

RWD ranges from accessibility in a mobile phone, tablet, laptop and larger desktop screens. There are factors to consider when building a fully responsive, from favouring units in em or rem than px, using responsive images(SVGs) to structuring with fluid layouts such as Flexbox and Grid to Media Queries. Also, you may want to know there are two school of thoughts about what view to build first, with most developers going for mobile firsr rather than desktop first.
